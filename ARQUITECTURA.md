## Flujo TГӯpico de Uso

```
1. LANDING PAGE
   в””в”Җ Visitante ve pГЎgina pГәblica
      в””в”Җ Clic en "Iniciar SesiГіn"

2. LOGIN PAGE
   в””в”Җ Usuario ingresa email/password
      в””в”Җ Clic en "Iniciar SesiГіn"

3. FIREBASE AUTH
   в””в”Җ Validar credenciales
      в”ңв”Җ вң… Гүxito вҶ’ crear JWT token
      в””в”Җ вқҢ Error вҶ’ mostrar mensaje

4. DASHBOARD
   в””в”Җ useAuth() obtiene usuario
      в””в”Җ AuthContext actualizado

5. NAVEGAR
   в””в”Җ Usuario puede navegar segГәn su rol
      в”ңв”Җ Admin: Acceso total
      в”ңв”Җ Doctor: Acceso a pacientes/estudios
      в”ңв”Җ Technician: Acceso a imГЎgenes
      в””в”Җ Patient: Acceso a datos propios

6. OPERACIГ“N (ej: lectura de paciente)
   в””в”Җ Clic en paciente
      в””в”Җ patients-service.getPatient(id)
         в””в”Җ Firestore Rules Check
            в”ңв”Җ вң… Autorizado вҶ’ mostrar datos
            в””в”Җ вқҢ Denegar вҶ’ error permission
```
